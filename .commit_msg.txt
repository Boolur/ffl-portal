Historical CSV import + Push-to-Service batch actions

Two related features for seeding and operating the Lead Distribution system:

Feature A: Historical CSV import with User Name matching
- Add "Assigned LO Name" to CsvUploadModal column mapper. When mapped, a
  new Step 3 "Map CSV names to portal users" appears: distinct names are
  auto-matched (exact name / exact email normalized, with ambiguity
  surfaced), admin picks a user or "Leave unassigned" per name, and
  decisions persist to localStorage so repeat imports auto-fill.
- bulkCreateLeadsBatch now accepts { assignment: { nameToUserId,
  fireBonzo } }. Matched rows get assignedUserId + assignedAt + status
  ASSIGNED; unmatched stay in the Unassigned Pool. Bonzo webhook firing
  is off by default (historical leads are usually already in Bonzo) but
  can be toggled on.
- New shared lib src/lib/leadNameMatch.ts: normalizeUserName (handles
  "Last, First" swap, punctuation, casing) + buildNameIndex + matchUser
  for deterministic client+server matching.

Feature B: Service registry + batch Push to Service
- New IntegrationService Prisma model (slug, name, type, active, config)
  with migration that seeds a Bonzo row. Admin CRUD at
  /admin/leads/services with archive/restore and gated hard delete.
- Service dispatcher in src/lib/services: pluggable serviceHandlers map
  (bonzo wraps forwardLeadToBonzo with structured skip/fail reasons),
  runPushBatch with bounded concurrency, summarizeBatch for per-reason
  UI grouping.
- pushLeadsToService server action (admin-only) executes a batch against
  a chosen service and returns { succeeded, skipped, failed } with
  per-lead reason details (no_assignee, no_webhook_url, http_error, etc).
- LeadsCRM's "Push to Service" button is now live (was "Coming soon"):
  opens PushToServiceModal which lists active services and shows a
  detailed result breakdown with success/skipped/failed grouping.

Notes:
- Instant Bonzo push on new-lead ingestion was already wired via
  distributeLead -> forwardLeadToBonzo and is unchanged.
- Lead schema field added: none (existing assignedUserId / assignedAt /
  status are reused).
